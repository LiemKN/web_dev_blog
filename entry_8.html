Redux

Redux is a predicatble state container for Javascript apps. It maintains the state of an entire application in an immutable state tree. Since it's immutable, when something changes, a new state tree is created using actions and reducers. To change a state, an action needs to be dipatched where the action is a Javascript object. The following are examples of actions. 

          { type: 'ADD_TODO', text: 'Go to swimming pool' }
          { type: 'TOGGLE_TODO', index: 1 }
          { type: 'SET_VISIBILITY_FILTER', filter: 'SHOW_ALL' }

To link the state and actions together, you need a reducer. The reducer is a function that takes the state and action and returns the next state that should result from the action. The following are examples of reducer functions. 

          function visibilityFilter(state = 'SHOW_ALL', action) {
            if (action.type === 'SET_VISIBILITY_FILTER') {
              return action.filter
            } else {
              return state
            }
          }
          
          function todos(state = [], action) {
            switch (action.type) {
              case 'ADD_TODO':
                return state.concat([{ text: action.text, completed: false }])
              case 'TOGGLE_TODO':
                return state.map((todo, index) =>
                  action.index === index
                    ? { text: todo.text, completed: !todo.completed }
                    : todo
                )
              default:
                return state
            }
          }

Some benefits of Redux include, predictability of outcome because the store provides clarity on how to link the current state with other parts of the application, organization because Redux is strict about how code should be organized, and ease of testing most Redux code is small and isolated.
